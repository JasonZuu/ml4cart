# Dynamics Response Prediction

## Purpose

This experiment predicts CAR-T/PDO response from time-lapse cell dynamics. It
combines:

- trajectory sequence features,
- track-level summary features,
- optional PDO size, antigen, immune, and stromal metadata.

The main model is `CrossAttnFusionModel`, which fuses sequence and track tokens
with TME metadata through cross-attention.

## Inputs

Default paths:

- Sequence dataset: `../dynamics_data/generated/trajectory_dataset_100.npz`
- Track dataset: `../dynamics_data/generated/track_dataset.npz`
- Split JSON: `../dynamics_data/data_split.json`

The `.npz` files must contain:

- `X`: feature array.
- `track_ids`: IDs used to match sequence and track arrays.

The split JSON should contain `train`, `val`, and optional `test*` split keys.
For JSON-only synthetic or external runs, labels can be provided through:

```json
{
  "train": ["CaseA"],
  "val": ["CaseB"],
  "meta": {
    "label_by_case": {"CaseA": 0, "CaseB": 1},
    "pdo_size": {"CaseA": 120.0},
    "antigen": {"CaseA": 45.0}
  }
}
```

For the full research dataset, labels are resolved from the annotation files
configured in `dynamics_model/config.py`.

## Demo Data

`demo_data/` contains two synthetic tracks and a JSON split/meta file:

```text
demo_data/
  generated/
    trajectory_dataset_100.npz
    track_dataset.npz
  data_split.json
```

Each `.npz` contains `X` and `track_ids`; `data_split.json` uses `DemoCaseA`
for train and `DemoCaseB` for val, with response labels, size-change metadata,
and TME features. Because there are only two demo cases, disable class
weighting:

```bash
python -m dynamics_model.train \
  --seq-path dynamics_model/demo_data/generated/trajectory_dataset_100.npz \
  --track-path dynamics_model/demo_data/generated/track_dataset.npz \
  --split-json dynamics_model/demo_data/data_split.json \
  --output-dir /tmp/ml4cart_demo_dynamics \
  --training-method ce \
  --use-class-weight false \
  --max-epochs 1 \
  --batch-size 2 \
  --hidden-size 4 \
  --fusion-size 4 \
  --use-wandb false \
  --seed 1
```

## Train

Default focal-loss training:

```bash
python -m dynamics_model.train \
  --seq-path ../dynamics_data/generated/trajectory_dataset_100.npz \
  --track-path ../dynamics_data/generated/track_dataset.npz \
  --split-json ../dynamics_data/data_split.json \
  --output-dir dynamics_model/results \
  --training-method focal \
  --learning-rate 1e-3 \
  --batch-size 256 \
  --max-epochs 200 \
  --hidden-size 16 \
  --fusion-size 32 \
  --dropout 0.3 \
  --use-wandb false \
  --seed 1
```

Cross-entropy training:

```bash
python -m dynamics_model.train \
  --training-method ce \
  --seq-path ../dynamics_data/generated/trajectory_dataset_100.npz \
  --track-path ../dynamics_data/generated/track_dataset.npz \
  --split-json ../dynamics_data/data_split.json \
  --output-dir dynamics_model/results/ce \
  --use-wandb false
```

Use W&B logging explicitly:

```bash
python -m dynamics_model.train \
  --use-wandb true \
  --wandb-project organoid-analyzer \
  --wandb-entity <entity>
```

Load best hyperparameters from a W&B sweep:

```bash
python -m dynamics_model.train \
  --use-wandb true \
  --use-wandb-best true \
  --wandb-project organoid-analyzer \
  --sweep-name grid_val_v1 \
  --metric-name val/best_auc
```
